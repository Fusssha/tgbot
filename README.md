# 👕 Fashion Shop Telegram Bot

Полнофункциональный Telegram бот для продажи одежды с поддержкой криптовалютных платежей (Bitcoin, Tron, TON).

## 🎯 Возможности

### Для пользователей:
- ✨ Просмотр каталога товаров по городам
- 🛒 Оформление заказов в несколько кликов
- 💳 Оплата криптовалютой (Bitcoin, Tron, TON)
- 👤 Личный профиль с контактной информацией
- 📋 История заказов
- 💬 Техническая поддержка

### Для администраторов:
- 📊 Статистика (количество пользователей, заказов, выручка)
- 🏙️ Добавление городов
- 📦 Добавление товаров в города с описанием и ценой
- 👥 Управление магазином

## 📋 Требования

- Python 3.9+
- pip (менеджер пакетов Python)
- Telegram аккаунт
- Telegram Bot Token (от @BotFather)

## 🚀 Установка

### 1. Клонирование репозитория или скачивание файлов

```bash
# Убедись, что у тебя есть все файлы:
# - main.py
# - config.py
# - database.py
# - handlers.py
# - keyboards.py
# - states.py
# - requirements.txt
# - .env.example
```

### 2. Установка зависимостей

```bash
pip install -r requirements.txt
```

### 3. Конфигурация

#### Получить Bot Token:
1. Напиши боту @BotFather в Telegram
2. Выполни команду `/newbot`
3. Следуй инструкциям, скопируй полученный токен

#### Узнать свой ID:
1. Напиши боту @userinfobot в Telegram
2. Скопируй свой User ID

#### Настройка .env файла:
```bash
# Копируем файл примера
cp .env.example .env

# Редактируем .env и добавляем:
BOT_TOKEN=твой_токен_от_BotFather
ADMIN_ID=твой_telegram_ID
```

### 4. Запуск бота

```bash
python main.py
```

Если все правильно настроено, ты увидишь в консоли:
```
2024-01-10 12:34:56,789 - __main__ - INFO - 🚀 Бот запущен...
```

## 💾 База данных

Проект использует SQLite для хранения данных:

### Таблицы:
- **users** - информация о пользователях
- **cities** - города с товарами
- **products** - товары в каждом городе
- **orders** - заказы пользователей
- **user_profiles** - дополнительная информация профиля

База данных создается автоматически при первом запуске.

## 🎮 Использование

### Для обычного пользователя:

1. **Поиск товаров:**

   - Выбери город

   - Нажми "⚙️ Админ панель" (доступно только админам)

2. **Добавление города:**
   - Нажми "➕ Добавить город"
   - Введи название города

3. **Добавление товара:**
   - Нажми "➕ Добавить товар"
   - Выбери город
   - Введи название, цену и описание

4. **Просмотр статистики:**
   - Нажми "📊 Статистика"
   - Посмотри количество пользователей, заказов и выручку

## 🔐 Криптовалютные платежи

### ⚠️ ВАЖНО: Это ДЕМО с фейковыми адресами!

Адреса в bot_config.py это примеры. Для реальной интеграции нужно:

2. **Для Tron:**
   - Использовать TronWeb или TronGrid
   - Мониторить смарт-контракты
   - Проверять входящие транзакции

3. **Для TON:**
   - Интегрировать TON API
   - Использовать wallet contracts
   - Реализовать webhook для подтверждений

## 🛠️ Как добавить реальные платежи

### Пример для Bitcoin (с API):

```python
import requests

async def check_bitcoin_payment(address, expected_amount):
    """Проверить платеж Bitcoin"""
    url = f"https://blockchain.info/address/{address}?format=json"
    
    try:
        response = requests.get(url)
        data = response.json()
        
        total_received = data['total_received'] / 1e8  # В BTC
        
        if total_received >= expected_amount:
            return True
        return False
    except Exception as e:
        logger.error(f"Bitcoin payment check error: {e}")
        return False
```

## 🐛 Решение проблем

### Ошибка: "ModuleNotFoundError: No module named 'aiogram'"

```bash
pip install aiogram==3.3.0
```

### Ошибка: "Invalid Bot Token"

- Проверь токен в .env файле
- Убедись, что скопировал токен правильно
- Попробуй получить новый токен у @BotFather

### Ошибка: "UserWarning: unclosed transport"

Это нормальное предупреждение при остановке бота (Ctrl+C), можешь игнорировать.

### БД не обновляется

Убедись, что:
- Файл shop.db находится в одной папке с скриптом
- У тебя есть права на запись в папку
- Нет конфликтов доступа (БД не открыта в другом приложении)

## 📊 Развертывание на сервере

### Вариант 1: Linux сервер (VPS)

```bash
# Подключиться по SSH
ssh user@your_server

# Установить Python
sudo apt-get install python3.9 python3-pip

# Скопировать файлы
scp -r bot_files user@your_server:/home/user/

# Установить зависимости
cd /home/user/bot_files
pip3 install -r requirements.txt

# Запустить в фоне (используя screen или nohup)
nohup python3 main.py &
```

### Вариант 2: Docker

```dockerfile
FROM python:3.9-slim

WORKDIR /app

COPY requirements.txt .
RUN pip install -r requirements.txt

COPY . .

CMD ["python", "main.py"]
```

```bash
docker build -t fashion-bot .
docker run -e BOT_TOKEN="your_token" fashion-bot
```

### Вариант 3: Heroku (бесплатно)

```bash

heroku login

# Создать приложение
heroku create your-app-name

# Установить переменную окружения
heroku config:set BOT_TOKEN="your_token"

# Развернуть
git push heroku main
```
